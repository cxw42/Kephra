 1. Preliminary note

 - Welcome to Kephra. This text editor is made to match your individual needs.
   If you have any comments, suggestions ans thanks, please visit our site at
   kephra.sf.net. More welcome you find in the main menu under Help > Welcome.

 - This is a so called stable version for the daily use. It contains mostly well
   tested features and bugs will be fixed as soon as possible with the next patch
   level, which will add no new features. If you like to help us or / and use 
   always the freshest features, please consider to use test versions, which have
   a three digit version number. Four digits mark nightly versions, in case you
   really believe in: "no risk no fun".


 2. News summary
    + search bar on the bottom.
    + first 2 extensions
    + much better drag 'n drop support
    + 10 doc spanning bookmarks and goto last edit
    + context menu on selected text and tabbar, which has 2 new optional icons
      and movable tabs.
    + "recent files" and "insert template" menus
    + menu and toolbar items can be checked and respond to events and states.
    + all menus and bar can modified via simple config files
    + menu labels have German umlauts
    + autosave, autonotify and blockformat
    + rename file and open all files of a dir


 3. News in detail

 - Searchbar
    + Diese sichtbarste und nützlichste Neuerung findet ihr am unteren Rand, über
    der Statuszeile. Sie enthält eine Textzeile, die man anklicken oder mit
    <Ctrl>+<F> anwählen kann (und damit auch wieder zurückspringen kann). Fängt
    man dann an einen Begriff dort einzugeben, wird bereits nach der Eingabe
    gesucht (inkrementelle Suche). Das war schon vom Dialog bekannt nur konnte der
    das Gefundene verdecken, die Suchzeile nicht.
    + Es ist jetzt auch möglich, einen markierten Text in die Textfelder der Such-
    zeile und des Suchdialoges mit der Maus zu ziehen. (Drag'n Drop)
    + In der Combobox kann man auch mit den Tasten <Hoch> und <Runter> bereits
    verwendete Suchbegriffe auswählen.
    + Folgende Tastaturkombinationen tun das gleiche wie im Textfeld und dienen
    der Navigation: <Bildhoch>, <Bildrunter>, beides auch mit <Ctrl> zum wechseln
    des Dokumentes sowie <Ctrl>+<Cursor Keys>.
    + <Enter> bzw. <Shift>+<Enter> sucht nach dem nächsten oder vorigen Vorkommen
    des Suchbegriffes. Diese Funktionen bringen auch die beiden Knöpfe rechts der
    Kombobox. <Ctrl>+<Enter> läßt zum ersten Fundort, +<Shift> zum letzten 
    springen.
    + Der dritte aktiviert das neue "goto last edit" (<Ctrl>+<Shift>+<G>),
    welches den Cursor an die Stelle springen lässt, an dem dieser Text zuletzt
    geändert wurde.
    + Der letzte Knopf ruft den bekannten Suchdialog, der alle Such- und Ersetzen-
    funktionen beinhaltet. Das Ändern der Suchoptionen geht aber auch mit einem
    Kontextmenü das sich bei Rechtsklick über der Suchzeile öffnet.
    + Sie kann auch mit dem roten X ganz links zugeklappt werden und wird sich
    mit dem nächsten Aufruf wieder öffnen. Ein Verlassen mit <Ctrl>+<Q> schließt
    auch.

 - Bookmarks and more Navigation
    Wenn es nicht genügt, daß der Editor den Ort der letzten Änderung speichert,
    verwendet Lesezeichen. Derer gibt es insgesamt 10, denn ihr Name entspricht
    den Ziffern 0 bis 9. Mit <Ctrl>+<Shift>+<Ziffer> wird das Lesezeichen in der
    aktuellen Zeile gesetzt und erscheint als blauer Pfeil links neben der Zeilen-
    nummer. Wenn nicht, sollte man im Menü "Ansicht" den "Markerrand" aktivieren.
    Falls unter dieser Ziffer bereits ein anderes Lesezeichen bestand, wurde es
    soeben hierher bewegt und falls es sich hier befand, wurde es soeben gelöscht.
    <Ctrl>+<Ziffer> bewegt den Cursor zum Dokument und der Stelle des Lesezeichens.
    + Das Blockweise navigieren mit dem Cursor hab ich auf <Alt>+<Bildhoch>
    (<Bildrunter>) verlegt. Textblöcke werden von Leerzeilen getrennt.
    + Neu ist auch die Klammernavigation die mit <Alt>+Richtungstasten bedient
    wird. Dabei kann man mit den vertikalen Richtungstasten zwischen den Klammern
    gleicher Art springen, mit den Horizontalen zwischen Klammern aller Art.
    + Mit <Alt>+<Shift>+<Richtungstasten> kann man allerdings nicht Text zwischen
    Klammern markieren sondern eine rechteckige Textauswahl treffen, wie es vorher
    nur möglich war, wenn man <Alt> gedrückt hält und mit der Maus auswählt.
    + <shift>+<Rück> Zieht Leerzeichen in Tabulatorbreite ein.
    + Das Springen zum zuletzt sichtbaren Dokument wurde aus <Ctrl>+<Rück> verlegt.

 - File menu
    Diese Menü erhielt mit 2 neuen Funktionen und 2 neuen Untermenüs die meisten
    Erweiterungen.
    + Das Untermenü Templates zählt selbt definierte Textbausteine auf, die man
    somit einfügen kann. Die zugehörige Datei wird unter file > templates bestimmt
    und kann über den Menüeintrag Einstellungen > Templates geöffnet werden.
    + Damit das Dateimenü nicht zu lang wird, habe ich 3 "Öffnen"-Funktionen in
    ein Untermenü gegeben. Darin befindet sich auch eine Funktion zum öffnen aller
    Dateien eines Verzeichnisses, die sicher noch erweitert werden kann.
    + "Zuletzt Verwendete" zählt die zuletzt geschlossenen Dateien auf, die hier
    wieder geöffnet werden können. file > history > length bestimmt die Länge
    dieses Menüs.
    + In der "Speichern"-Gruppe findet sich jetzt eine Funktionen, die auch mit
    <Ctrl>+<Alt>+<Shift>+<S> aufrufbar ist, und das Umbenennen der gerade offenen
    Datei ermöglicht.
    + Auch das Untermenü für Dateigruppen ist um 2 Funktionen reicher. Es kann 
    nun eine alternative Dateigruppe (session) unter einem in der Hauptconfig
    angegebenen Dateinamen (file > session > backup) gespeichert und wieder 
    hergestellt werden, ohne jedes mal den Dateinamen eingeben zu müssen.

 - Drag 'n Drop
     Sicher konnte man schon in Version 0.3 Dateien und Verzeichnisse öffnen, in
     dem man sie über die Reiterleiste zog und dort losließ. Da dies aber keine
     wirklich große Fläche ist, mag dies vielen nicht aufgefallen sein. Jetzt kann
     man zum Glück dies über der gesamten Fläche des Editors tun, was für mich auch
     eine große Erleichterung ist.

 - Formating Text
    + Im Untermenü "Formatieren" sind 4 neue Einträge die 2 neue Funktionen des
    Editors verwenden. Das ist zum einen "Zeilen umbrechen". Im Gegensatz zum
    rein optischen Zeilenumbruch, werden hier Zeilen, die eine bestimmte Länge
    überschreiten, wirklich in mehrere ausgeteilt. "Blockformat" hingegen "füllt"
    auch halbleere Zeilen mit der nächsten auf und erzeugt so massive Textblöcke,
    deren linke Begrenzung sich bei aktivierter autoeinrückung an der Einrückung 
    der ersten Zeile orientiert. Beide Funktionen gibt es im in den Varianeten:
    a) Begrenzung am rechten Rand und b) formatiere auf eine frei wählbare Breite.

 - Context menus and bars
    Nicht nur über der neuen Suchzeile findet ihr neue Kontextmenüs wenn ihr die
    rechte Maustaste drückt:
    + Das wichtigste ist vielleicht das Menü das sich bei markiertem Text zeigt.
    Durch sinnvolles Aufteilen der Funktionen in 2 Menüs konnte ich nicht nur
    mehr Funktionen einbringen, sondern es bieten sich zumeist nur die Funktionen
    an, die in der Situation auch sinnvoll sind.
    + In die Reiterleiste können nun die Reiter auch bewegt werden. Voerst nur mit
    den Tasten <Strg>+<BildHoch>|<BildRunter>.
    + Neu sind auch die 2 optionalen Icons in der Reiterleiste je ganz links
    (neues Dokument) und ganz rechts (Dokument schließen). Auch läßt sich mit dem
    globalen Einstellungsschlüssel app > tabbar > middle_click definieren, welche
    funktion die mittlere Maustaste über der Reiterleiste bekommt.
    + Das Menü über der Reiterleiste listet alle offenen Dokumente mit dem vollen
    Pfad der zugehörigen Dateien auf und man kann hier auch das aktuelle Dokument
    wechseln.
    + Die Kontextmenüs über den Zellen der Statuszeile sind jetzt auch alle nur
    noch mit dem  Rechtsklick sichtbar zu machen. Ein Linksklick schaltet die
    Funktioen an oder aus oder wechselt zwischen den wichtigsten Einstellungen.
    + Die zweite Zelle der Statuszeile zeigt übrigens jetzt auch die Anzahl der
    markierten Zeilen an.

 - Configs
    The new menus and bars are useful, but the real change from version 0.3 to
    0.4 is the replacement of XRC with homegrown, more powerfull technology.
    + Thatswhy menus can not contain german umlauts.
    + Also 
    Auch einzelne Einträge können mit einem Häckchen (ja/nein) oder Kreis
    (Auswahl) versehen sein werden, oder auch deaktiviert werden. Knöpfen in
    Werkzeugleisten können einen eingedrückten Zustand haben.
    + Neu ist auch die in dem Menü Einstellungen > Benutzerschnittstellen 
    auffindbare Kommandoliste, in der bestimmt wird, welcher interne Befehl, 
    Tastaturkombination und Symbol zu zu einem Kommando gehört. Auch welche 
    Ereignisse ihn deaktivieren und welche internen Funktionen seinen Zustand
    bestimmen, wird hier festgelegt. Welches Etikett und welcher Hilfetext zu ihm
    gehört steht in den Sprachdateien (auch im Menü Einstellungen), die nun die
    gesamte Übersetzung des Editors in eine Sprache beinhalten. Wenn ihr ihn in
    eine übersetzen wollt meldet euch ruhig bei mir für Hilfe.
    + Unter Einstellungen > Benutzerschnittstellen findet ihr auch die neuen 
    Dateien in YAML, mit denen ihr ganz leicht die Menüs und Leisten ändern könnt. 
    Für vorübergehendes kommentieren einzelner Einträge benutzt bitte eine Raute 
    (#) am Ende der Zeile.
    + in den Haupteinstellungen könnt ihr unter app > window > title nun auch die
    Titelleiste verändern. Welche Informationen, die sich aktualisieren werden,
    ihr auch dazu verwenden könnt, seht ihr in config/global/default/default.conf
    + Die im Titel angezeigten Pfade sind jetzt auch immer OS-konform.
    
 - First 2 Extensions
    + With <Ctrl>+<F4> or the icon in the tool bar you can open or close the 
      notepad panel on the right side. There you can leave text that always is
      visible, even after a restart. <F4> let the cursor jump between the editor
      and the notepad and <F5> does a perl "eval" with the content of the notepad.
      The output will be shown in the output panel.
    + Is the cursor at the edit panel active you can run with <F5> the current
      file as a Perl script and stop that any time with <Shift>+<F5>. <Ctrl>+<F5>
      opens or closes the output panel.
    + All settings regarding these panel like color, font size or behavior can
      be found under app > panel.

 5. Hints

    + Read the "Advanced - Tour" in the "Help" menu. This is full of goodies,

    + To understand the main config file better, look into the defaults under:
    kephra/config/global/default/default.conf. It explains many keys and tells
    you about possible settings.
